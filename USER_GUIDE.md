## Introduction

Stock Picker is a comprehensive algorithmic trading platform that enables you to:

- **Create and manage multiple portfolios** with different strategies
- **Build custom trading strategies** using technical factors (RSI, MACD, Moving Averages)
- **Backtest strategies** on historical data to validate performance
- **Execute automated trades** through Alpaca broker integration
- **Monitor real-time performance** with detailed analytics
- **Receive alerts** for important trading events

### Key Concepts

**Portfolio**: A collection of stocks and cash managed together with a specific strategy.

**Strategy**: An algorithmic trading approach built by combining technical factors with custom weights.

**Factor**: A technical indicator (like RSI or MACD) that generates buy/sell signals based on market data.

**Backtest**: Simulation of strategy performance using historical price data.

**Signal**: A recommendation (BUY, SELL, or HOLD) generated by your strategy.

## Building Trading Strategies

Strategies are the brain of your trading system. Stock Picker uses a **composable factor system** where you combine multiple technical indicators.

### Available Factors

**1. RSI (Relative Strength Index)**
- **Purpose**: Identifies overbought/oversold conditions
- **Parameters**:
  - `period`: Lookback period (default: 14)
  - `oversold`: Buy threshold (default: 30)
  - `overbought`: Sell threshold (default: 70)
- **Best for**: Mean reversion strategies

**2. MACD (Moving Average Convergence Divergence)**
- **Purpose**: Identifies momentum and trend changes
- **Parameters**:
  - `fast`: Fast EMA period (default: 12)
  - `slow`: Slow EMA period (default: 26)
  - `signal`: Signal line period (default: 9)
- **Best for**: Trend-following strategies

**3. Moving Average Crossover**
- **Purpose**: Captures trend direction changes
- **Parameters**:
  - `short`: Short MA period (default: 50)
  - `long`: Long MA period (default: 200)
- **Best for**: Long-term trend strategies

### Creating a Strategy

1. Navigate to **Strategies** page
2. Click **"Create Strategy"**
3. Configure basic settings:
   - **Name**: "My Momentum Strategy"
   - **Portfolio**: Select target portfolio
   - **Description**: Strategy objectives

4. **Add Factors**:
   - Click "Add Factor"
   - Select factor type (RSI, MACD, MA Crossover)
   - Set weight (0.0 to 1.0)
   - Configure parameters
   - Repeat for multiple factors

5. **Configure Risk Management**:
   - **Stop Loss**: Percentage loss to trigger exit (e.g., 5%)
   - **Take Profit**: Target gain to lock in profits (e.g., 15%)
   - **Position Size**: Portfolio % per trade (e.g., 10%)
   - **Max Positions**: Maximum simultaneous holdings (e.g., 10)
   - **Max Portfolio Risk**: Total portfolio exposure limit

6. **Save Strategy**

### Example Strategy Configurations

**Conservative Growth Strategy**
```
Factors:
- MA Crossover (weight: 0.5, short: 50, long: 200)
- RSI (weight: 0.5, period: 14, oversold: 35, overbought: 65)

Risk Management:
- Stop Loss: 3%
- Take Profit: 10%
- Position Size: 8%
- Max Positions: 8
```

**Aggressive Momentum Strategy**
```
Factors:
- MACD (weight: 0.4, fast: 12, slow: 26, signal: 9)
- RSI (weight: 0.3, period: 14, oversold: 25, overbought: 75)
- MA Crossover (weight: 0.3, short: 20, long: 50)

Risk Management:
- Stop Loss: 7%
- Take Profit: 20%
- Position Size: 15%
- Max Positions: 12
```

### Understanding Factor Weights

Weights determine the importance of each factor in the final decision:

- Weights must sum to 1.0
- Higher weight = more influence
- Example: RSI (0.4) + MACD (0.3) + MA (0.3) = 1.0

The strategy combines factor scores using weighted average to produce a final signal.

## Backtesting Your Strategies

Backtesting validates strategy performance using historical data before risking real capital.

### Running a Backtest

1. Go to **Backtests** page
2. Click **"New Backtest"**
3. Configure:
   - **Strategy**: Select strategy to test
   - **Start Date**: Beginning of test period
   - **End Date**: End of test period
   - **Initial Capital**: Starting cash (e.g., $10,000)
   - **Commission**: Cost per trade (e.g., $1)
   - **Slippage**: Expected price slippage (e.g., 0.1%)

4. Click **"Run Backtest"**
5. Wait for results (processing time varies)

### Interpreting Results

**Performance Metrics**:
- **Total Return**: Overall profit/loss percentage
- **Sharpe Ratio**: Risk-adjusted return (higher is better)
- **Max Drawdown**: Largest peak-to-trough decline
- **Win Rate**: Percentage of profitable trades
- **Profit Factor**: Ratio of gross profits to gross losses
- **Average Win/Loss**: Mean profit/loss per trade

**Equity Curve**:
- Visualizes portfolio value over time
- Shows drawdown periods
- Indicates consistency of returns

**Trade Analysis**:
- All trades executed during backtest
- Entry/exit prices and dates
- P&L for each trade
- Signals that triggered trades

### What Makes a Good Backtest?

✅ **Good Indicators**:
- Sharpe ratio > 1.0
- Win rate > 50%
- Profit factor > 1.5
- Consistent returns (smooth equity curve)
- Max drawdown < 20%

⚠️ **Warning Signs**:
- Few trades (< 30) - insufficient data
- Extremely high returns - likely overfitting
- Large gaps in trading activity
- High correlation to single stock

### Backtest Comparison

Compare multiple backtests to find the best strategy:

1. Go to **Backtests → Compare**
2. Select 2-4 backtests
3. Review side-by-side metrics
4. Analyze equity curves
5. Choose winning strategy

## Live Trading

### Enabling Automated Trading

1. Navigate to **Strategy Detail** page
2. Ensure strategy is backtested successfully
3. Toggle **"Enable Automated Trading"**
4. Confirm risk parameters

### How Automated Trading Works

**Schedule**:
- Strategies evaluate during market hours (9:30 AM - 4:00 PM ET)
- Runs every 15 minutes by default
- Skips weekends and market holidays

**Execution Flow**:
1. Strategy fetches latest price data
2. Evaluates all factors for each stock
3. Generates signals (BUY/SELL/HOLD)
4. Checks risk constraints
5. Submits orders to Alpaca broker
6. Logs trade and sends notification

### Manual Trading

Execute trades manually from:

**Stock Detail Page**:

3. Enter quantity or dollar amount
4. Confirm order

**Portfolio Page**:
1. View current positions
2. Click **"Close Position"** to sell
3. Confirm exit

### Trade History

View all executed trades on **Trades** page:

- Filter by portfolio, strategy, or date range
- See entry/exit prices and P&L
- Review signal reasoning
- Export trade data

## Best Practices & Tips

### Strategy Development

**1. Start Simple**
- Begin with 1-2 factors
- Use well-tested indicators (RSI, MA Crossover)
- Validate with backtest before adding complexity

**2. Diversify Factors**
- Combine momentum and mean-reversion
- Use different timeframes
- Balance technical and fundamental signals

**3. Backtest Thoroughly**
- Test on at least 1 year of data
- Use multiple market conditions (bull, bear, sideways)
- Verify minimum 30+ trades in backtest

**4. Avoid Overfitting**
- Don't optimize to perfection on historical data
- Use simple parameter values
- Test on out-of-sample data

**5. Monitor and Adjust**
- Review strategy performance weekly
- Disable underperforming strategies
- Gradually refine parameters based on live results

### Risk Management

**Position Sizing**:
- Never allocate more than 10-15% per position
- Maintain 20% cash reserve
- Scale position size with conviction

**Stop Losses**:
- Always use stop losses (recommended: 5-7%)
- Adjust based on stock volatility
- Don't move stop losses lower (only higher for profits)

**Diversification**:
- Hold 5-10 positions minimum
- Diversify across sectors
- Don't concentrate in correlated stocks

**Circuit Breakers**:
- Set max daily loss limit (e.g., 3% of portfolio)
- Pause trading during unusual volatility
- Review before resuming

### Performance Optimization

**1. Factor Selection**
- Not all factors work for all stocks
- Test factor performance on specific stocks
- Weight factors based on proven success

**2. Timeframe Alignment**
- Match factor periods to your trading horizon
- Short-term: 10-20 day periods
- Long-term: 50-200 day periods

**3. Data Quality**
- Ensure price data is complete
- Check for gaps or anomalies
- Use adjusted prices (account for splits/dividends)

**4. Transaction Costs**
- Factor in commissions and slippage
- Higher turnover = higher costs
- Optimize holding period

## Troubleshooting

### Common Issues

**Problem: Strategy not generating signals**

Possible causes:
- Strategy not enabled for automated trading
- No stocks meet factor criteria
- Risk limits preventing new positions
- Price data not available

Solutions:
- Check strategy status on detail page
- Review factor thresholds (may be too strict)
- Verify risk management settings
- Check stock data dashboard for completeness

**Problem: Backtest fails or shows errors**

Possible causes:
- Insufficient historical data for date range
- Invalid factor parameters
- Database connection issue

Solutions:
- Reduce backtest date range
- Verify all factor parameters are valid
- Check stock data availability for period
- Retry after a few minutes

**Problem: Trades not executing**

Possible causes:
- Alpaca broker connection issue
- Insufficient cash in portfolio
- Market closed
- Order rejected by broker

Solutions:
- Check portfolio cash balance
- Verify market hours (9:30 AM - 4:00 PM ET)
- Review broker error message in trade history
- Check Alpaca account status

**Problem: Real-time updates not working**

Possible causes:
- WebSocket connection dropped
- Authentication expired
- Network connectivity issue

Solutions:
- Check connection indicator (top right)
- Refresh page to reconnect
- Re-login if session expired
- Check browser console for errors

**Problem: Missing price data for stocks**

Possible causes:
- Stock recently added
- Data provider API limit reached
- Stock delisted or inactive

Solutions:
- Wait for next data sync (runs hourly)
- Check stock data dashboard
- Verify stock is actively traded
- Contact support if persistent

## Appendix: Glossary

**Alpha**: Excess return of an investment relative to a benchmark

**Beta**: Measure of volatility relative to the market

**Drawdown**: Decline from peak to trough in portfolio value

**Factor**: Technical indicator used to generate trading signals

**P&L (Profit & Loss)**: Net gain or loss on a position or portfolio

**Position**: Holdings of a specific stock in a portfolio

**Sharpe Ratio**: Risk-adjusted return metric (return per unit of risk)

**Signal**: Trading recommendation (BUY, SELL, or HOLD)

**Slippage**: Difference between expected and actual execution price

**Strategy**: Algorithm combining multiple factors to make trading decisions

**Trade**: Execution of a buy or sell order

**Win Rate**: Percentage of trades that are profitable
